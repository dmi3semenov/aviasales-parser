# Aviasales Parser — контекст для Claude

## Что делает проект
Парсит Aviasales через Playwright (headless Chromium), собирает все варианты рейсов для заданных комбинаций дат и строит многостраничный Excel со сводками и аналитикой.

## Архитектура

```
trip-config.json          ← единственный источник параметров
       ↓
generate-trips.js         ← генерирует комбинации дат и URL Aviasales
       ↓  (--run)
aviasales-parser.js       ← открывает браузер, парсит каждый URL, сохраняет xlsx/json/png
       ↓
build-summary.js          ← читает все свежие xlsx из output/, строит итоговый Excel
```

## Быстрый старт (весь флоу)

```bash
# 1. Посмотреть комбинации (без запуска)
node generate-trips.js

# 2. Запустить парсинг (откроется браузер — решить капчу на первой странице)
node generate-trips.js --run

# 3. После завершения парсинга — собрать сводку
node build-summary.js
```

## trip-config.json — что можно менять

```json
{
  "маршрут": {
    "города":    ["MOW", "MSQ", "MOW"],   // IATA-коды: origin → dest → origin
    "названия":  ["Москва", "Минск", "Москва"],  // отображается в заголовках и имени файла
    "пассажиры": 2
  },
  "даты": {
    "окна_вылета": [
      { "от": "29.04", "до": "02.05" },   // DD.MM — окно дат вылета
      { "от": "07.05", "до": "10.05" }
    ]
  },
  "ночи": { "мин": 2, "макс": 3 },
  "бюджет_макс": 27000,                  // порог "не берём" на вкладке Итог (в рублях)
  "критерии": {
    "MOW_MSQ": "любой",
    "MSQ_MOW": "любой"
  }
}
```

Для маршрута с промежуточной остановкой A→B→C→A: раскомментируй `"остановки"` в секции `для_маршрута_с_пересадкой` (пример там же).

## Вкладки итогового Excel (в порядке слева направо)

| Вкладка | Цвет | Описание |
|---------|------|----------|
| **Итог** | 🟠 оранжевая | Главный лист: берём / чуть дорого но можно / не берём — по порогу `бюджет_макс`. Лучшие варианты с ценой на человека и советом по времени вылета. Вывод по каждому окну. |
| **По времени вылета** | 🟡 жёлтая | 2D-сетка: строки = слот вылета туда (Ночь/Утро/День/Вечер), столбцы = слот вылета обратно → мин. цена. Цветовая шкала. "Лучший →" и "Лучший ↓". |
| **Рек - Карточки** | 🟢 зелёная | Детальные карточки на каждую (дата + ночи), отсортированные по цене. Цвет-рейтинг по слотам, совет. |
| **Рек - Таблица** | 🟢 зелёная | Все варианты в одной таблице, сортировка по цене. Столбцы: вылет, возврат, ночи, мин цена, кол-во билетов, лучший слот, разброс, совет. |
| **Рек - По дате** | 🟢 зелёная | То же что Рек - Таблица, но сортировка по дате вылета (хронологически). |
| **Рек - По окнам** | 🟢 зелёная | Анализ по окнам вылета: сетка дата × ночи, лучший вариант на каждый день, итог окна. |
| По датам | — | Матрица дата вылета × ночи → мин. цена. |
| Все варианты | — | Все рейсы по возрастанию цены, с деталями. |
| Все сводки | — | Сырые данные из каждого отдельного поиска. |
| Пояснения | — | Расшифровка всех вкладок и терминов. |

## Как работает парсинг

- Playwright открывает браузер (НЕ headless — нужна капча)
- Первая страница — ждёт 60 секунд для ввода капчи
- Затем проходит по всем URL по очереди (~10–15 сек на каждый)
- На каждый URL сохраняет: `as_DDMM-DDMM_MOW-MSQ-MOW_MM-DD_HH-MM.xlsx/.json/.png`

## Логика build-summary.js

- Берёт только **последний** файл для каждой комбинации дат (по алфавиту/времени в имени)
- Поэтому можно запускать парсинг повторно — старые файлы в output/ не мешают
- Имя итогового файла: `as_СВОДКИ_{Город1}-{Город2}_{MM-DD}_{HH-MM}.xlsx`

## Фильтрация цен
- Цена должна содержать `₽`
- Диапазон: 5 000 — 300 000 ₽ (отсекает мусор из DOM)

## Текущие зависимости
```json
{ "playwright": "^1.x", "exceljs": "^4.x", "xlsx": "^0.18.x" }
```
Установить: `npm install && npx playwright install chromium`
